# Manual Testing & Quality Assurance Portfolio

This repository contains comprehensive manual testing reports and UI/UX evaluations conducted on live production environments. These projects demonstrate my ability to perform exploratory testing, document defects, categorize bugs by severity, and provide actionable Quality Management System (QMS) insights.

## 📂 Project 1: Amrutam Web & Mobile App - Bug Report
**Platform:** Web (Desktop) & Mobile (iOS/Android)  
**Tools Used:** Manual Exploratory Testing, Bug Tracking Documentation  

**Overview:**
Conducted end-to-end exploratory testing on the Amrutam Ayurveda platform[cite: 2]. Identified and documented a total of 30 defects across 5 core functional areas[cite: 2].
*   **Total Bugs Logged:** 30 (Web: 12, Mobile: 18)[cite: 2]
*   **Severity Breakdown:** High (11), Medium (11), Low (8)[cite: 2]
*   **Key Modules Tested:** Onboarding, Doctor Consultation, E-commerce (Search/Cart), Content Forum, and User Profile[cite: 2].
*   **Critical Findings:** Identified high data-loss bugs during Web Onboarding, unexplained user blocks in Doctor Consultation, and critical checkout failures[cite: 2]. Documented exact reproduction steps, expected vs. actual results, and attached visual evidence[cite: 2].

## 📂 Project 2: AIVOI.AI - Manual Testing & QMS Insights
**Platform:** Web (B2B SaaS Platform)  
**Tools Used:** Functional Testing, UI/UX Evaluation, Form Validation Testing  

**Overview:**
Evaluated the AIVOI.AI platform through manual testing, focusing on website navigation, form validations, UI consistency, and overall user experience.
*   **Key Findings:** Uncovered critical input validation vulnerabilities, specifically in the Phone, Name, and Company Name fields (accepting unrestricted alphanumeric inputs).
*   **UX Optimization:** Proposed workflow optimizations for module navigation to reduce friction for first-time users browsing the site. 
*   **QMS Deliverable:** Compiled a structured Quality Management System insight report detailing functional issues, UX observations, and actionable recommendations for input validation and error handling.
